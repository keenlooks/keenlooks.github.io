CI quality gate for the site, run by `.github/workflows/site-ci.yml` on every push to master and every PR (the site is built with `JEKYLL_ENV=production` inside `ruby:2.7`, using `_config.ci.yml` to empty `site.url` so the build is self-contained).
- `check-inline-scripts.js`: extracts every inline `<script>` from the built HTML and runs `node --check` on it, catching the production HTML-compression bug where a `//` comment swallows the rest of the script (CLAUDE.md gotcha #1).
- `check-links.js`: resolves every internal `href`/`src` against `_site` (tries the path, `path/index.html`, `path.html`; strips `?query` and `#fragment`), failing on missing targets; known dynamic paths go in its `ALLOWLIST` array.
- `smoke.js`: serves `_site` on :8080 and opens each page in `pages.json` with headless Chromium, failing on uncaught page errors and warning (only) on blank canvases; it needs `npm i --no-save playwright http-server` and `npx playwright install --with-deps chromium` first.
Run them locally against a built `_site` with `node ci/check-inline-scripts.js _site`, `node ci/check-links.js _site`, `node ci/smoke.js _site`; test fixtures for the two checkers live in `ci/_fixtures/` (underscore-prefixed so Jekyll never copies them into the built site).
